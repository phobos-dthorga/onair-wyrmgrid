# Governance

OnAir WyrmGrid currently uses a benevolent-maintainer model. Phobos A. D'thorga
sets product direction, release readiness, branding, and security policy while
the contributor community shapes implementation through issues, discussions,
reviews, and proposals.

Plugin protocol, storage compatibility, security boundaries, licensing, and
project identity changes require maintainer approval and an architecture
decision record. Routine fixes and documentation can proceed through normal
review.

Governance will be revisited before the first stable release or when sustained
maintainer participation makes a broader model practical.
