# OnAir WyrmGrid v0.2.0

WyrmGrid v0.2.0 is an early prerelease that expands the original foundation
into a connected flight-operations workspace. It remains read-only at external
provider boundaries and does not invent facts when a provider omits them.

## Highlights

- MSFS 2024 telemetry through the isolated SimConnect provider, including the
  visible bridge-link ritual, recording lifecycle, automatic-start preference,
  honest disconnect gaps, and per-measurement display units.
- Hoard flight history and whole-flight debrief graphs with long-recording
  downsampling, fuel and attitude evidence, SimBrief planned-versus-recorded
  comparisons, and planned/recorded Atlas routes.
- Atlas regional boundaries, zoom-sensitive detail, airport weather evidence,
  FBO and fleet context, historical tracks, and Dispatch route projection.
- Read-only OnAir Staff workspace with verified qualifications plus shared
  search, filtering, sorting, result-count, and drill-down presentation tools.
- Core authorization, Security Centre, permission lifetimes, encrypted
  SQLCipher storage, operating-system credential storage, and password-encrypted
  portable backup and restore.
- Persistent OnAir credentials and optional SimBrief Pilot ID storage at the
  user's discretion, without placing end-user keys in builds or CI.
- Themed launch artwork, configurable motion, low-resource launch options, and
  improved dialog navigation.

## Installation and compatibility

The CI-built installers preserve WyrmGrid's product identity and per-user
installation scope. Close WyrmGrid and run the newer setup over an existing
installation; do not uninstall first. The release workflow verifies a clean
installation or an in-place upgrade from the nearest published NSIS release.

Device-local database and credential keys are never embedded in release
artifacts. A password-encrypted portable backup is recommended before upgrading
an important prerelease installation or moving to another computer.

## Prerelease boundary

Packages remain unsigned prerelease artifacts until platform and updater signing
policies are complete. The GitHub release is created as a draft for checksum,
installation, startup, licence, and offline-behaviour verification before manual
publication.
