# Audio provider authoring and packaging

Audio Capture Providers are external native processes. They implement the
versioned framing in
[Audio Capture Provider protocol version 2](audio-capture-provider-protocol.md)
and are delivered as independently installable `.wyrmaudio` packages. A
provider can be written in any language that can read and write the documented
standard streams and produce a native executable for its target platform.

## Package layout

Package format version 1 is a deterministic ZIP envelope:

```text
example.wyrmaudio
├── wyrmgrid-package.json
├── audio-provider.json
└── bin/example-audio-provider.exe
```

`wyrmgrid-package.json` is generated by the packager. It records kind
`audio_provider`, the package identity/version, the fixed manifest path, and an
exact size and SHA-256 inventory. `audio-provider.json` uses
`schemas/audio-provider-manifest-v2.schema.json`; its ID and version must match
the package. `entry_point` is a safe portable relative stem: its exact path must
appear for declared Linux/macOS targets and the same path with `.exe` appended
must appear for Windows x86-64.

Use a reverse-domain provider ID and version the provider independently from
WyrmGrid. An immutable `(audio_provider, id, version)` may never refer to
different archive contents. Publish a new provider version for every changed
binary or manifest.

## Build a package

From the repository root:

```powershell
npm run audio-provider:package -- `
  --source path\to\package-root `
  --output path\to\example.wyrmaudio
```

The source directory must contain `audio-provider.json` and its declared entry
point. `--include SOURCE=PACKAGE_PATH` can add files from another build output;
`--force` explicitly replaces an existing output. The packager rejects unsafe
paths, links, empty or oversized files, duplicate case-insensitive paths,
missing entry points, invalid identities, and archives outside the common
package limits.

The repository's synthetic reference artifact is produced with:

```powershell
npm run audio-provider:distribution
```

That artifact exercises packaging and protocol conformance only. It does not
capture a real device and must not be described as a supported audio provider.

## What installation authorizes

Installation copies validated bytes into WyrmGrid's managed version store. It
does not:

- select or start the executable;
- request microphone or output-device permission;
- enable WyrmGrid's audio recording consent;
- choose any source or codec profile; or
- grant access to OnAir credentials, simulator telemetry, plugins, recordings,
  or the network.

The user separately enables the package, selects it, enables audio recording,
reviews any operating-system permission prompt, and chooses exact sources.
Providers report capabilities and source facts; WyrmGrid owns policy,
retention, encryption, source selection, and capture timing.

## Lifecycle and failure rules

WyrmGrid can keep several packages installed but uses at most one selected audio
provider. A selected provider must be enabled and compatible with the current
platform. Active recording blocks provider replacement, disable, rollback, and
removal. Disabling or removing the selected provider clears the saved selection.

At startup the process must return a hello descriptor matching the installed
manifest's ID, name, version, platform, and capabilities. Any mismatch,
unsupported platform, missing executable, failed launch, timeout, invalid frame,
or protocol violation makes audio unavailable without blocking simulator
telemetry or the rest of WyrmGrid.

## Trust boundary

Package format version 1 has no publisher signature. The archive digest and
exact inventory detect accidental or deliberate byte changes before install but
do not establish who wrote the code or make native code safe. Authors should
publish source, reproducible build instructions, package digests, supported
platforms, device/privacy behaviour, and a clear vulnerability-reporting path.
WyrmGrid will add signing, revocation, stronger launch integrity, and sandboxing
only through separately versioned security decisions.
